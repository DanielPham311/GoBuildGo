## Naming Conventions

| Category | Convention | Example |
|---|---|---|
| Files (components) | PascalCase | `ComponentCard.tsx` |
| Files (utilities) | kebab-case | `style-engine.ts` |
| Files (routes) | kebab-case | `component-card.tsx` |
| Components | PascalCase | `ComponentCard` |
| Functions | camelCase | `calculateStyleScore` |
| Constants | UPPER_SNAKE_CASE | `MAX_BUDGET` |
| DB tables (SQL) | snake_case | `setup_items` |
| DB fields (Prisma) | camelCase | `setupId` |
| API routes | kebab-case | `/api/v1/affiliate-clicks` |
| CSS classes | Tailwind utilities | `flex items-center gap-4` |
| Custom CSS | `@layer components` | `.btn-primary { ... }` |
| Type interfaces | PascalCase | `StyleResult` |
| Enums | PascalCase | `ComponentCategory` |
| Enum values | PascalCase | `ComponentCategory.Desk` |
| Zod schemas | camelCase + Schema | `setupSchema` |
| Test files | `*.test.ts` or `*.spec.ts` | `style-engine.test.ts` |

## Module Boundaries (Unidirectional)

```
src/types/          в†ђ leaf (no src imports)
    в†‘
src/lib/            в†ђ imports: types
    в†‘
src/db/             в†ђ imports: lib, types
    в†‘
src/server/         в†ђ imports: db, lib, types
    в†‘
src/stores/         в†ђ imports: lib, types, server/actions
    в†‘
src/components/     в†ђ imports: lib, stores, types
    в†‘
src/app/            в†ђ imports: components, lib, stores, types
```

**Rules:**
1. Never import "up" the dependency graph
2. Use barrel exports (`index.ts`) per module
3. Server Actions are the boundary between client and server
4. No cross-component imports (planner/ doesn't import from components/)
5. Shared types live in `src/types/`, not duplicated
